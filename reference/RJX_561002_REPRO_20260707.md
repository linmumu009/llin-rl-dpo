## Current Interpretation

- Earlier synthetic full-param test failed with memory error `207001`, working operator `aclnnFlashAttentionScore`.

The visual tower is not the direct trigger in this text-only config:

- The copied config has `attr.images: null`.
- FSDP `apply_modules` include language model and `lm_head`, not `model.visual`.
- The failure occurs in language-model rotary backward.

However, pure text SFT/DPO should still freeze unrelated visual parameters unless a multimodal task requires them, because earlier 4096 long-answer tests showed full-param training has substantially higher memory pressure.

## Current Root-Cause Hypothesis

The best current explanation is:

1. The native NPU rotary backward op `aclnnRotaryPositionEmbeddingGrad` has a tiling limit in `RopeHalfGradSetTiling`.
2. In the colleague-matched path, original row 20 tokenizes to length `2506` with `2052` supervised labels.
3. The NPU tiling code aligns that shape to `reserveAlignNum = 2592`.
4. `2592` exceeds the supported aicore tiling range, so the op fails with error `561002`.
5. Truncating to `cutoff_len=2048` avoids the error; changing micro batch size or disabling activation offload does not.

The unexplained part is why rows 16-31 as a single first step passes, while row 20 after one or more completed optimizer steps fails. That suggests stateful execution details such as autograd graph scheduling, kernel cache/match behavior, dynamic-shape tiling cache, or NPU stream/runtime state may be involved. It does not change the first failing operator: synchronous launch confirms the failure is inside `npu_rotary_mul_backward`.
