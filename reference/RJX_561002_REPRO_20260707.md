## Data Localization

The tokenized full dataset has 25167 rows. Only two rows reach the hard `4096` token limit, so the failure is not simply "any 4096-token sample fails".

With `shuffle: false`, `BaseRandomBatchSampler` uses rank-strided sampling. For world size 8 and micro batch 2, iteration 2 rank 4 receives original rows:

```text
20 and 28
```

Their tokenized lengths are:

```text
row 20: input_len 2506, labels 2052
row 28: input_len 1264, labels 707
```

The failing tiling value `reserveAlignNum = 2592` is consistent with an aligned value derived from this rank-local maximum sequence length, but we have not proven the exact tiling formula yet.

## Current Interpretation

The failure is real and reproducible in our isolated container when we match the colleague source, env, config, and data. It is not the same issue as our earlier full-param synthetic OOM:

- Earlier synthetic full-param test failed with memory error `207001`, working operator `aclnnFlashAttentionScore`.
- The colleague-matched run fails in native NPU rotary backward tiling, `aclnnRotaryPositionEmbeddingGrad`, error `561002`.

The visual tower is not the direct trigger in this text-only config:

- The copied config has `attr.images: null`.
- FSDP `apply_modules` include language model and `lm_head`, not `model.visual`.
- The failure occurs in language-model rotary backward.

However, pure text SFT/DPO should still freeze unrelated visual parameters unless a multimodal task requires them, because earlier 4096 long-answer tests showed full-param training has substantially higher memory pressure.
